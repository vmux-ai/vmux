# Zoom Pane Implementation Plan (VMX-112)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement tmux-style "zoom pane" — `<leader> z` (chord `Ctrl+G, z`, already wired as `PaneCommand::Zoom`) toggles the focused leaf pane to fill the tab's main area, hiding sibling panes.

**Architecture:** A `Zoomed { leaf, hidden: Vec<Entity> }` component on the `Tab` entity holds zoom state per-tab. A new system applies `Display::None` to siblings on the ancestor path; an `on_remove` component hook restores `Display::Flex` when the component is removed. Auto-unzoom is triggered by pane navigation, splits, or pane removal. A new `is_zoomed` field on `TabsHostEvent` drives a "Z" pill in the Dioxus header chrome.

**Tech Stack:** Rust, Bevy ECS, Dioxus (WASM), rkyv for IPC events.

---

## Spec

See `docs/specs/2026-05-11-zoom-pane-design.md`. Note: the spec uses tmux's `Focus{Up/Down/Left/Right}` terminology. The actual command variants in the codebase are `PaneCommand::SelectLeft/SelectRight/SelectUp/SelectDown` (see `crates/vmux_command/src/command.rs:229-300` — selection is what tmux calls focus). This plan uses the real variant names.

```rust
fn siblings_to_hide(world: &World, leaf: Entity, tab: Entity) -> Vec<Entity> {
    let mut result = Vec::new();
    let mut cur = leaf;
    while cur != tab {
        let Some(parent) = world.get::<ChildOf>(cur).map(|p| p.0) else {
            break;
        };
        if world.get::<PaneSplit>(parent).is_some()
            && let Some(children) = world.get::<Children>(parent)
        {
            for &child in children.iter() {
                if child != cur {
                    result.push(child);
                }
            }
        }
        cur = parent;
    }
    result
}
```

fn tab_of(

    child_of_q: &Query<&ChildOf>,

) -> Option<Entity> {
    let mut cur = leaf;

fn collect_siblings_to_hide(
    leaf: Entity,
    tab: Entity,
    child_of_q: &Query<&ChildOf>,
    all_children: &Query<&Children>,
    split_dir_q: &Query<&PaneSplit>,
) -> Vec<Entity> {
    let mut result = Vec::new();

    while cur != tab {
        let Ok(parent) = child_of_q.get(cur).map(|p| p.0) else { break };
        if split_dir_q.get(parent).is_ok()

        {
            for &child in children.iter() {
                if child != cur {
                    result.push(child);
                }
            }
        }
        cur = parent;
    }
    result
}

- [ ] **Step 3: Register a `Pane` `on_remove` hook**

Add to `PanePlugin::build`, alongside the `Zoomed` hook from Task 4:

```rust
app.world_mut()
    .register_component_hooks::<Pane>()
    .on_remove(|mut world: DeferredWorld, ctx: HookContext| {
        let removed = ctx.entity;
        let mut tabs_to_clear: Vec<Entity> = Vec::new();
        let mut q = world.query::<(Entity, &Zoomed)>();
        for (tab, z) in q.iter(&world) {
            if z.leaf == removed {
                tabs_to_clear.push(tab);
            }
        }
        for tab in tabs_to_clear {
            world.commands().entity(tab).remove::<Zoomed>();
        }
    });
```

`world.query::<...>()` inside a `DeferredWorld` works — it borrows immutably while we collect, then `world.commands()` enqueues the removal which fires the `Zoomed` `on_remove` hook on the next deferred-flush, restoring `Display::Flex` on hidden siblings.
