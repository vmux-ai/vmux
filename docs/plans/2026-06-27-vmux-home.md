# vmux://home Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: implement task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. This plan is executed **inline / directly** (not via subagents): vmux CEF builds are huge and long-lived subagents drop sockets. Warm the build once, implement all tasks, defer the single runtime test to the end (the user runs it).

**Goal:** Add a `vmux://home` Google-style launcher page with full command-bar parity (live filter, path completion, history, pages/commands/tabs/spaces) by sharing the command bar's composition, and make it the default `startup_url`.

**Architecture:** Home is the command bar's body in a centered page shell instead of a modal. WASM: extract a shared `CommandPalette` component used by both the modal and home. Host: extract the payload builder so a new `on_home_data_request` observer replies with the same `CommandBarOpenEvent`; reuse the existing dispatch/path-complete/history events by adding `"home"` to their host lists.

**Tech Stack:** Rust, Bevy ECS, Dioxus (WASM), `bevy_cef` rkyv bin-IPC, Tailwind v4.

**Build/verify commands (this repo):**
- Native tests: `cargo test -p vmux_layout -p vmux_setting`
- WASM typecheck of pages: `cargo check -p vmux_server --target wasm32-unknown-unknown --features web`
- Warm full build (background, once): `cargo build -p vmux_desktop`
- Worktree: all paths below are under `.worktrees/vmux-home/`. Use this worktree's own `target/`; do not share `CARGO_TARGET_DIR`.

- [ ] **Step 2: Add `build_command_bar_open_payload` fn** — same file

Wraps the snapshot→Vec mapping (pages already include `agent_pages()` + history shortcut; commands via `command_list`; spaces from the snapshot) and the existing `command_bar_open_payload(...)`:

```rust
#[allow(clippy::too_many_arguments)]
pub(crate) fn build_command_bar_open_payload(
    open_id: u64,
    native_windowed: bool,
    space_name: String,
    current_url: String,
    spaces_snapshot: &CommandBarSpacesSnapshot,
    agents_snapshot: &CommandBarAgentsSnapshot,
    pages_snapshot: &CommandBarPagesSnapshot,
    active_stack_count: usize,
    tabs: Vec<CommandBarTab>,
    target: Option<vmux_command::open_target::OpenTarget>,
) -> CommandBarOpenEvent {
    let app_agent_entries: Vec<AppAgentEntry> = agents_snapshot
        .strategies
        .iter()
        .map(|s| AppAgentEntry {
            id: app_agent_id(&s.provider, &s.model),
            name: format!("New {}/{} chat (App)", s.provider, s.model),
        })
        .collect();
    let mut pages = pages_snapshot.pages.clone();
    pages.extend(agent_pages());
    let history_shortcut = command_shortcut("browser_open_history");
    if !history_shortcut.is_empty()
        && let Some(page) = pages.iter_mut().find(|page| page.host == "history")
    {
        page.shortcut = history_shortcut;
    }
    let bar_commands: Vec<CommandBarCommandEntry> = command_list(app_agent_entries)
        .into_iter()
        .map(|e| CommandBarCommandEntry { id: e.id, name: e.name, shortcut: e.shortcut })
        .collect();
    let bar_spaces = spaces_snapshot
        .spaces
        .iter()
        .map(|s| {
            let is_active = s.id == spaces_snapshot.active_space_id;
            CommandBarSpace {
                id: s.id.clone(),
                name: s.name.clone(),
                profile: s.profile.clone(),
                is_active,
                tab_count: if is_active { active_stack_count as u32 } else { 0 },
            }
        })
        .collect();
    command_bar_open_payload(
        open_id, native_windowed, space_name, current_url, bar_spaces, tabs, bar_commands, target, pages,
    )
}
```

Model the entity-reply on `on_path_complete_request` (handler.rs:1624-1640). The observer builds the payload via the Task 2 helpers and emits `COMMAND_BAR_OPEN_EVENT` back to the requesting webview entity with `target: InPlace`.

```rust
use bevy::prelude::*;
use bevy_cef::prelude::{BinEventEmitterPlugin, BinHostEmitEvent, BinReceive};
use vmux_command::event::COMMAND_BAR_OPEN_EVENT;
use vmux_command::open_target::OpenTarget;

use crate::command_bar::handler::{
    TabGatherParams, build_command_bar_open_payload, gather_command_bar_tabs,
};
use crate::home::event::HomeDataRequest;
use crate::settings::EffectiveStartupUrl;
use vmux_command::event::{
    CommandBarAgentsSnapshot, CommandBarPagesSnapshot, CommandBarSpacesSnapshot,
}; // re-exported in command_bar; adjust path to wherever the snapshots are defined

pub struct HomePlugin;

impl Plugin for HomePlugin {
    fn build(&self, app: &mut App) {
        app.world_mut().spawn(crate::home::PAGE_MANIFEST);
        app.add_plugins(BinEventEmitterPlugin::<(HomeDataRequest,)>::for_hosts(&["home"]))
            .add_observer(on_home_data_request);
    }
}

fn on_home_data_request(
    trigger: On<BinReceive<HomeDataRequest>>,
    tab_gather: TabGatherParams,
    spaces_snapshot: Res<CommandBarSpacesSnapshot>,
    agents_snapshot: Res<CommandBarAgentsSnapshot>,
    pages_snapshot: Res<CommandBarPagesSnapshot>,
    stack_q: Query<Entity, With<crate::stack::Stack>>,
    mut commands: Commands,
) {
    let webview = trigger.event().webview;
    let tabs = gather_command_bar_tabs(&tab_gather);
    let active_stack_count = stack_q.iter().count();
    let space_name = spaces_snapshot.active_space_name.clone();
    let payload = build_command_bar_open_payload(
        0, false, space_name, String::new(),
        &spaces_snapshot, &agents_snapshot, &pages_snapshot,
        active_stack_count, tabs, Some(OpenTarget::InPlace),
    );
    commands.trigger(BinHostEmitEvent::from_rkyv(webview, COMMAND_BAR_OPEN_EVENT, &payload));
}
```
