# vmux_cli Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create `vmux_cli` crate that ships a single `vmux` binary with `mcp` subcommand and a default action that launches Vmux.app, replacing the standalone `vmux_mcp` binary.

**Architecture:** New crate `crates/vmux_cli` with `[[bin]] name = "vmux"`. Uses `clap` derive for parsing. `mcp` subcommand delegates to `vmux_mcp::protocol::run_stdio` (`vmux_mcp` becomes lib-only). Default action launches Vmux.app via macOS `open -a Vmux`; Linux returns an "unsupported" error. App launching uses a small `AppLauncher` trait so the unit test can record calls without spawning a process.

**Tech Stack:** Rust 2024, clap 4 (derive), tokio (workspace dep, for async `mcp`), assert_cmd 2 (dev-dep for integration tests), predicates 3 (dev-dep).

**Spec:** `docs/specs/2026-05-08-vmux-cli-design.md`

Insert below the existing `run` fn, before `#[cfg(test)]`:

```rust
pub struct OpenAppLauncher;

impl AppLauncher for OpenAppLauncher {
    #[cfg(target_os = "macos")]
    fn launch(&self, app_name: &str) -> io::Result<()> {
        let status = std::process::Command::new("open")
            .arg("-a")
            .arg(app_name)
            .status()?;
        if status.success() {
            Ok(())
        } else {
            Err(io::Error::other(format!(
                "open -a {app_name} exited with {status}"
            )))
        }
    }

    #[cfg(not(target_os = "macos"))]
    fn launch(&self, _app_name: &str) -> io::Result<()> {
        Err(io::Error::new(
            io::ErrorKind::Unsupported,
            "launching the Vmux app is not supported on this platform yet",
        ))
    }
}
```
