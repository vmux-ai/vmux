# vmux status bar (Dioxus web)

Rust + **Dioxus** web app served from loopback by `vmux` (see `vmux_webview::StatusBarHostedPlugin`) and loaded in the active pane’s CEF strip.

- **`src/main.rs`** — `dioxus::launch` entry.
- **`src/app.rs`** — root **App** component (status strip markup).
- **`src/bridge.rs`** — `document::eval` script for clock + `window.cef.listen("vmux_status", …)`.
- **`src/payload.rs`** — host payload types and `apply_payload`.
- **`assets/index.html`** — static shell for `wasm-bindgen` output (tracked).
- **`assets/input.css`** — Tailwind entry (`@tailwind` + `@layer base`).
- **`tailwind.config.js`** — theme (`tmux` colors, `text-status`, font stack).
- **`assets/status.css`** — Tailwind **build output** (included with `include_str!` in `app.rs`). **Not committed**; regenerated by `build.rs` / `npm run build:css`.
- **`package.json`** — `tailwindcss` CLI for `build:css`.
- **`dist/`** — **not** hand-edited: produced by `build.rs` (WASM + JS from `wasm-bindgen` + shell `index.html`).

## Build (`dist/`)

Requires **Node.js** (Tailwind) and the **`wasm-bindgen` CLI** on `PATH`, on the **same semver** as the `wasm-bindgen` dependency (see `Cargo.toml` / `Cargo.lock`, same idea as `vmux_history`):

```bash
cargo install wasm-bindgen-cli --version 0.2.115 --locked
```

**Styles:** after editing `src/**/*.rs` (Tailwind classes) or `assets/input.css` / `tailwind.config.js`, the next native build runs Tailwind as needed.

From the repo root (`build.rs` runs `npm`, wasm32 release, `wasm-bindgen`, optional `wasm-opt`, copies `assets/index.html` when `dist/` is stale):

```bash
env -u CEF_PATH cargo build -p vmux_status_bar
```

`vmux` embeds `crates/vmux_status_bar/dist/` via Axum `ServeDir`; **`index.html`** must exist there.

Set **`VMUX_STATUS_UI_URL`** (e.g. a static dev server URL) to skip the embedded bundle for that session.
